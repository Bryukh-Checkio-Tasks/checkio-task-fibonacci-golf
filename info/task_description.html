The Fibonacci numbers or Fibonacci sequence are the numbers in the following integer sequence:
0, 1, 1, 2, 3, 5, 8, 13, 21...
But it's not only one interesting integer sequence. There are many various sequences like the Fibonacci numbers, where each elements calculated with previous elements. Let's look and use several of them.

Here are described several integer sequence which you should implement:

  • fibonacci: 
    f(0)=0, f(1)=1, f(n)=f(n-1)+f(n-2)
  • tribonacci: 
    f(0)=0, f(1)=1, f(2)=1, f(n)=f(n-1)+f(n-2)+f(n-3)
  • lucas: 
    f(0)=2, f(1)=1, f(n)=f(n-1)+f(n-2)
  • jacobsthal: 
    f(0)=0, f(1)=1, f(n)=f(n-1)+2*f(n-2)
  • pell: 
    f(0)=0, f(1)=1, f(n)=2*f(n-1)+f(n-2)
  • perrin: 
    f(0)=3, f(1)=0, f(2)=2, f(n)=f(n-2)+f(n-3)
  • padovan: 
    f(0)=0, f(1)=1, f(2)=1, f(n)=f(n-2)+f(n-3)

You are given a name of sequence and a number "n". You should find the n-th element for the given sequence.

In this mission the main goal to make your code as short as possible. But not a code as is it. We will check a length of the compiled code. The shorter your compiled code, the more points you earn. You can use the local checker to know the compiled size locally. Your score for this mission is dynamic and directly related to the length of your code. For reference, scoring is based on the number of characters used. 1000 bytes is the maximum allowable and it will earn you zero points. For each byte less than 1000, you earn 1 point. For example for 300 byte long code earns you 700 points.

Input: Two arguments. A name of a sequence as a string and a number "n" as a positive integer.

Output: The n-th element of the sequence as an integer.

How it is used: In this mission, you can learn how to code in DRY (don't repeat yourself) way.

Preconditions: 10 ≤ n < 400
